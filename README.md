# Тестовое Задание

Написать погодное веб-приложение на .Net Framework, использовать можно любые библиотеки, помогающие решить поставленную задачу. В приложении должен быть интерфейс и бекенд.

## UI

- Отобразить один экран с погодной информацией: текущая, почасовая (показывать оставшиеся часы из текущего дня и все часы следующего), прогноз погоды на 3 дня.
- Обработать показ загрузки и ошибку, если что-то пошло не так, с кнопкой повторного запроса.
- По дизайну никаких ограничений нет, все на ваш вкус.

## Геолокация и запросы

- Геолокацию зафиксировать на использовании города Москва.
- Данные получать из запросов API:

  `http://api.weatherapi.com/v1/current.json?key=fa8b3df74d4042b9aa7135114252304&q=LAT,LON`

  `http://api.weatherapi.com/v1/forecast.json?key=fa8b3df74d4042b9aa7135114252304&q=LAT,LON&days=3`

Реализация графической составляющей на усмотрение кандидата, оформление должно быть понятным, не запутанным, но соответствовать ТЗ.

## Общее описание проекта

Это небольшое погодное приложение про Москву.

Сделано как обычное .NET-решение с разделением на слои:
- `WeatherApp.Web` - веб-интерфейс (страница с текущей погодой, почасовым и дневным прогнозом).
- `WeatherApp.Api` - API-слой.
- `WeatherApp.Application` - бизнес-логика, валидация и маппинг.
- `WeatherApp.Domain` - доменные модели.
- `WeatherApp.Infrastructure` - работа с внешним погодным API и парсинг данных.
- `tests/*` - набор unit-тестов по основным частям.

Логика простая: приложение запрашивает прогноз у WeatherAPI, преобразует ответ в свои модели и отдает в UI.

По ТЗ сейчас используется фиксированная геолокация Москвы и прогноз на 3 дня.
